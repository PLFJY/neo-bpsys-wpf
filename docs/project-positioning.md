# 项目定位与术语

## 社区名称

本项目应继续保留“第五人格 BP 展示工具”或“BP display tool”的叫法。Identity V 民间赛社区已经默认理解这类工具不只显示 BP，还包括队伍管理、比分显示、赛后数据、前台输出窗口、OBS 捕获和导播侧辅助。

README 中的定位是：基于 .NET WPF、面向第五人格民间赛直播的 BP 画面直播展示工具，目标是帮助民间赛事获得更接近官方职业赛事的直播效果。

## 架构定位

从内部架构看，它也可以被描述为“非官方第五人格赛事直播导播辅助系统”。这个说法更适合开发文档，因为代码中存在多类功能：

| 能力 | 说明 |
| --- | --- |
| 后台控制 UI | 主 WPF 窗口，供导播/赛事工作人员操作 |
| 前台窗口 | 独立 WPF 输出窗口，供 OBS 或其他直播软件捕获 |
| 数据中心 | 队伍、对局、角色、比分、计时器、全局 Ban 等共享状态 |
| 引导式 BP | 根据 `GameRule.json` 逐步导航到后台页面并高亮操作区域 |
| SmartBP/OCR | 目前成熟的是赛后数据 OCR 自动回填；全流程自动 BP/自动切屏仍是 TODO |
| 插件系统 | 允许第三方扩展后台页面、前台窗口、注入控件和服务 |

## 后台与前台

本仓库中的“Frontend”“FrontedWindow”“前台窗口”不是 Web 前端。

| 术语 | 代码含义 |
| --- | --- |
| 后台 / backend | 主 WPF 控制窗口，主要是 `MainWindow`、后台页面和 ViewModel |
| 后台页面 | `Views/Pages/*` 下的 WPF `Page`，通过 WPF-UI `NavigationView` 展示 |
| 前台窗口 / FrontedWindow | `Views/Windows/*` 下的独立 WPF 输出窗口，用于 OBS 捕获 |
| 宿主 | 主应用 `neo-bpsys-wpf`，负责构建 Host、注册服务、加载插件 |

维护时不要引入 Web frontend 的假设，例如路由、浏览器 DOM、CSS 构建链等。这里的前台输出是 WPF 窗口、Canvas、绑定和资源字典。

## 用户场景

典型使用者是民间赛组织者、导播或直播技术人员：

1. 在后台导入或维护队伍、选手、角色和当前对局。
2. 打开 BP、过场、比分、赛后数据、小组件等前台窗口。
3. 在 OBS 中用窗口捕获或游戏捕获采集这些前台窗口。
4. 通过后台页面推进 BP、比分、地图 BP、赛后数据等状态。
5. 必要时用插件扩展额外页面、窗口或控件。

因此，改动时要优先考虑导播现场的稳定性：少弹阻断窗口、保留已有布局配置、保持重启/升级路径可预期。
